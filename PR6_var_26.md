## **Часть 1**

Компания **BudgetControl** занимается управлением бюджетами различных подразделений организации.   
Ежедневно обрабатываются данные о бюджетах отделов: часть подразделений укладывается в лимит, часть — превышает бюджет, а часть — требует пересмотра финансирования.  
Руководство хочет контролировать подразделение "маркетинг", отслеживать выделенные лимиты и выявлять потенциальные перерасходы.

Ваша задача как RPA-разработчика — автоматизировать работу с данными о бюджетах через **DataFrame** в **Puzzle RPA**, отфильтровать подразделения по названию, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

---

**Шаги:**

1. **Создать исходный файл `budgets.xlsx` с 10 записями:**

```python
# budgets.xlsx
| budget_id | department    | budget_limit | period_start | period_end | status     |
|-----------|---------------|--------------|--------------|------------|------------|
| B001      | "маркетинг"   | 500000       | 2026-03-01   | 2026-03-31 | "активен"  |
| B002      | "продажи"     | 750000       | 2026-03-01   | 2026-03-31 | "активен"  |
| B003      | "маркетинг"   | 300000       | 2026-02-01   | 2026-02-28 | "закрыт"   |
| B004      | "IT"          | 1200000      | 2026-03-01   | 2026-03-31 | "активен"  |
| B005      | "маркетинг"   | 450000       | 2026-01-01   | 2026-01-31 | "закрыт"   |
| B006      | "HR"          | 200000       | 2026-03-01   | 2026-03-31 | "активен"  |
| B007      | "маркетинг"   | 550000       | 2026-04-01   | 2026-04-30 | "планируется"|
| B008      | "продажи"     | 680000       | 2026-02-01   | 2026-02-28 | "закрыт"   |
| B009      | "маркетинг"   | 400000       | 2026-03-01   | 2026-03-31 | "активен"  |
| B010      | "IT"          | 950000       | 2026-02-01   | 2026-02-28 | "закрыт"   |
```

* **Типы данных:**

```python
budget_id: str
department: str  # "маркетинг", "продажи", "IT", "HR"
budget_limit: int  # лимит бюджета в рублях
period_start: date
period_end: date
status: str  # "активен", "закрыт", "планируется"
```

2. **Считать исходный файл `budgets.xlsx` в объект в DataFrame `budgets_df`**


3. **Добавить столбец `is_current` — флаг текущего периода:**

```python
budgets_df["is_current"] = (budgets_df["period_start"] <= today) & (budgets_df["period_end"] >= today)
```

4. **Отфильтровать подразделение "маркетинг":**

```python
marketing_budgets_df = FilterDataFrame(
    table=budgets_df,
    filter_condition='department == "маркетинг"'
)
```

5. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="marketing_budgets.xlsx")

# Уведомляем пользователя
NotifyUser(message="Бюджеты отдела маркетинга успешно сохранены в marketing_budgets.xlsx!")
```

## **Часть 2**

После фильтрации бюджетов маркетинга руководство хочет проанализировать фактические расходы.   
Существует второй файл `expenses.xlsx`, в котором указаны данные о расходах подразделений.   
Нужно объединить данные и выделить расходы, превышающие установленный лимит бюджета, а результат сохранить и показать пользователю.

---

```python
# expenses.xlsx

|------------|-----------|----------------|--------------|--------------|
| E001       | B001      | 480000         | 2026-03-10   | "реклама"    |
| E002       | B002      | 720000         | 2026-03-08   | "комиссии"   |
| E003       | B001      | 520000         | 2026-03-12   | "мероприятия"|
| E004       | B004      | 1100000        | 2026-03-09   | "оборудование"|

| E006       | B007      | 600000         | 2026-04-05   | "мероприятия"|
| E007       | B009      | 350000         | 2026-03-11   | "реклама"    |
| E008       | B001      | 495000         | 2026-03-14   | "контент"    |
```

* **Типы данных:**

```python
expense_id: str
budget_id: str
expense_amount: int  # сумма расхода в рублях
expense_date: date
expense_type: str  # "реклама", "мероприятия", "контент", "комиссии", "оборудование"
```

2. **Считать исходный файл `expenses.xlsx` в объект в DataFrame `expenses_df`**

3. **Объединить `marketing_budgets_df` и `expenses_df` по `budget_id` (inner join):**

```python
combined_df = JoinDataFrames(
    table1=marketing_budgets_df,
    table2=expenses_df,
    keys1=["budget_id"],
    keys2=["budget_id"],
    join_type="inner"
)
```

4. **Выбрать расходы, превышающие лимит бюджета:**

```python
overbudget_df = FilterDataFrame(
    table=combined_df,
    filter_condition='expense_amount > budget_limit'
)
```

5. **Сохранить результат в Excel и уведомить пользователя:**

```python

SaveExcel(file_path="overbudget_expenses.xlsx")

# Уведомляем пользователя
NotifyUser(message="Аналитика по расходам сверх бюджета сохранена в overbudget_expenses.xlsx!")
```
