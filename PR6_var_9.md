# Вариант 9 - SmartInventory: выявление дефицита товаров на складе

---

## **Часть 1**

Компания **SmartInventory** управляет складскими запасами товаров для розничной сети по городам России.   
На складе хранятся электроника, бытовая техника, товары для дома и другие категории.  
Руководство хочет контролировать товары с критически низким остатком, отслеживать продажи и выявлять позиции, требующие срочного пополнения.

Ваша задача как RPA-разработчика — автоматизировать работу с остатками через **DataFrame** в **Puzzle RPA**, отфильтровать товары по количеству, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

inventory_df["is_critical"] = inventory_df["quantity"] < 10

low_stock_df = FilterDataFrame(

## **Часть 2**

После фильтрации товаров с низким остатком руководство хочет проверить активность продаж.   
Существует второй файл `sales.xlsx`, в котором указаны данные о продажах за последнюю неделю.   
Нужно объединить данные и выделить товары с высоким спросом и низким остатком, а результат сохранить и показать пользователю.

---

1. **Создать файл `sales.xlsx` с 8 записями:**

```python
# sales.xlsx
| item_id | sales_last_week | sales_month | avg_daily_sales | stock_turnover |
|---------|-----------------|-------------|-----------------|----------------|
| I002    | 12              | 45          | 1.7             | 3.2            |
| I003    | 8               | 30          | 1.1             | 2.5            |
| I005    | 15              | 55          | 2.1             | 4.0            |
| I007    | 6               | 22          | 0.9             | 1.8            |
| I009    | 10              | 38          | 1.4             | 2.9            |
| I010    | 7               | 28          | 1.0             | 2.2            |
| I001    | 3               | 12          | 0.4             | 0.8            |
| I004    | 5               | 20          | 0.7             | 1.5            |
```

* **Типы данных:**

```python
item_id: str
sales_last_week: int
sales_month: int
avg_daily_sales: float
stock_turnover: float
```

2. **Считать исходный файл `sales.xlsx` в объект в DataFrame `sales_df`**

3. **Объединить `low_stock_df` и `sales_df` по `item_id`:**

```python
combined_df = JoinDataFrames(
    table1=low_stock_df,
    table2=sales_df,
    keys1=["item_id"],
    keys2=["item_id"],
    join_type="left"
)
```

4. **Добавить расчет `reorder_priority`:**

```python
combined_df["reorder_priority"] = combined_df["sales_last_week"] / combined_df["quantity"]
```

5. **Выбрать товары с продажами за неделю > 5:**

```python
high_demand_df = FilterDataFrame(
    table=combined_df,
    filter_condition='sales_last_week > 5'
)
```

6. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="high_demand_low_stock.xlsx")

# Уведомляем пользователя
NotifyUser(message="Аналитика по товарам с высоким спросом и низким остатком сохранена в high_demand_low_stock.xlsx!")
```
