## **Часть 1**

Компания **SupportDesk** занимается обработкой обращений пользователей в службу технической поддержки.   
В системе регистрируются тикеты разных статусов: открытые, в работе, решённые и закрытые.  
Руководство хочет контролировать открытые обращения, отслеживать приоритеты задач и выявлять тикеты, требующие срочного внимания.

Ваша задача как RPA-разработчика — автоматизировать работу с обращениями через **DataFrame** в **Puzzle RPA**, отфильтровать тикеты по статусу, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

---

2. **Считать исходный файл `tickets.xlsx` в объект в DataFrame `tickets_df`**


3. **Добавить столбец `days_open` — дней с создания тикета:**

```python
tickets_df["days_open"] = (today - tickets_df["created_date"]).days
```

## **Часть 2**

После фильтрации открытых обращений руководство хочет проанализировать распределение тикетов по сотрудникам поддержки.   
Существует второй файл `employees.xlsx`, в котором указаны данные о специалистах и их нагрузке.   
Нужно объединить данные и выделить обращения с высоким приоритетом, а результат сохранить и показать пользователю.

---

1. **Создать файл `employees.xlsx` с 8 записями:**

```python
# employees.xlsx
| emp_id | emp_name   | department | position     | active_tickets | avg_response_time | rating |
|--------|------------|------------|--------------|----------------|-------------------|--------|
| EMP01  | "Петрова А."| "поддержка"| "специалист"| 5              | 2.5               | 4.2    |

| EMP04  | "Кузнецова Е."| "поддержка"| "лид"     | 2              | 1.5               | 4.9    |
| EMP05  | "Смирнов П."| "поддержка"| "специалист"| 6              | 2.9               | 4.1    |
| EMP06  | "Фролова К."| "поддержка"| "старший"   | 4              | 2.1               | 4.5    |
| EMP07  | "Егоров В." | "поддержка"| "специалист"| 8              | 3.5               | 3.7    |
| EMP08  | "Лебедева Л."| "поддержка"| "менеджер" | 1              | 1.2               | 5.0    |
```

* **Типы данных:**

```python
emp_id: str
emp_name: str
department: str
position: str
active_tickets: int
avg_response_time: float  # часов
rating: float             # 1-5
```

2. **Считать исходный файл `employees.xlsx` в объект в DataFrame `employees_df`**

3. **Объединить `open_tickets_df` и `employees_df` по назначенному специалисту (добавим emp_id в tickets):**

```python
# Предварительно добавим emp_id в open_tickets_df для объединения
open_tickets_df["emp_id"] = ["EMP01", "EMP03", "EMP02", "EMP01", "EMP04", "EMP03", "EMP02", "EMP05"]

combined_df = JoinDataFrames(
    table1=open_tickets_df,
    table2=employees_df,
    keys1=["emp_id"],
    keys2=["emp_id"],
    join_type="inner"
)
```

4. **Добавить расчет `urgency_score`:**

```python
combined_df["urgency_score"] = combined_df["days_open"] * (3 if combined_df["priority"] == "высокий" else 1)
```

5. **Выбрать тикеты с высоким приоритетом:**

```python
high_priority_df = FilterDataFrame(
    table=combined_df,
    filter_condition='priority == "высокий"'
)
```

6. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="high_priority_tickets.xlsx")

# Уведомляем пользователя
NotifyUser(message="Аналитика по обращениям с высоким приоритетом сохранена в high_priority_tickets.xlsx!")
```
