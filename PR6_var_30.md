# Вариант 30 - BillingAnalytics: контроль выставленных счетов и неоплаченных счетов

---

## **Часть 1**

Компания **BillingAnalytics** занимается управлением выставленными счетами и контролем оплат от клиентов.   
Ежедневно обрабатываются данные о счетах: часть оплачена, часть — ожидает оплаты, а часть — просрочена.  
Руководство хочет контролировать счета с суммой более 20000 рублей, отслеживать статусы оплат и выявлять неоплаченные счета.

Ваша задача как RPA-разработчика — автоматизировать работу с данными о счетах через **DataFrame** в **Puzzle RPA**, отфильтровать счета по сумме, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

---

**Шаги:**

1. **Создать исходный файл `invoices.xlsx` с 10 записями:**

```python
# invoices.xlsx
| invoice_id | client_name   | amount  | invoice_date | due_date   | invoice_status |
|------------|---------------|---------|--------------|------------|----------------|
| INV001     | "ООО Альфа"   | 15000   | 2026-02-28   | 2026-03-15 | "оплачено"     |
| INV002     | "ИП Бета"     | 25000   | 2026-03-01   | 2026-03-16 | "не оплачено"  |
| INV003     | "ООО Гамма"   | 32000   | 2026-02-25   | 2026-03-12 | "не оплачено"  |
| INV004     | "ЗАО Дельта"  | 18000   | 2026-02-27   | 2026-03-14 | "оплачено"     |
| INV005     | "ИП Эпсилон"  | 45000   | 2026-02-20   | 2026-03-07 | "просрочено"   |
| INV006     | "ООО Альфа"   | 22000   | 2026-02-28   | 2026-03-15 | "не оплачено"  |
| INV007     | "ИП Бета"     | 35000   | 2026-03-02   | 2026-03-17 | "оплачено"     |
| INV008     | "ООО Гамма"   | 12000   | 2026-02-26   | 2026-03-13 | "оплачено"     |
| INV009     | "ЗАО Дельта"  | 28000   | 2026-03-01   | 2026-03-16 | "не оплачено"  |
| INV010     | "ИП Эпсилон"  | 50000   | 2026-02-25   | 2026-03-12 | "оплачено"     |
```

* **Типы данных:**

```python
invoice_id: str
client_name: str
amount: int  # сумма счета в рублях
invoice_date: date
due_date: date
invoice_status: str  # "оплачено", "не оплачено", "просрочено"
```

2. **Считать исходный файл `invoices.xlsx` в объект в DataFrame `invoices_df`**


3. **Добавить столбец `is_high_value` — флаг крупного счета:**

```python
invoices_df["is_high_value"] = invoices_df["amount"] > 20000
```

4. **Отфильтровать счета с суммой > 20000:**

```python
high_value_invoices_df = FilterDataFrame(
    table=invoices_df,
    filter_condition='amount > 20000'
)
```

5. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="high_value_invoices.xlsx")

# Уведомляем пользователя
NotifyUser(message="Крупные счета успешно сохранены в high_value_invoices.xlsx!")
```

2. **Считать исходный файл `clients.xlsx` в объект в DataFrame `clients_df`**

3. **Объединить `high_value_invoices_df` и `clients_df` по `client_name` (inner join):**

```python
combined_df = JoinDataFrames(
    table1=high_value_invoices_df,
    table2=clients_df,
    keys1=["client_name"],
    keys2=["client_name"],
    join_type="inner"
)
```

4. **Выбрать счета со статусом "не оплачено":**

```python
unpaid_high_df = FilterDataFrame(
    table=combined_df,
    filter_condition='invoice_status == "не оплачено"'
)
```
