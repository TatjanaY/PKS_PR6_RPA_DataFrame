---

## **Часть 1**

Компания **ITHelpDesk** занимается обработкой тикетов службы технической поддержки.   
Ежедневно поступают десятки обращений: часть относится к отделу "IT", часть — к другим подразделениям, а часть — требует эскалации.  
Руководство хочет контролировать тикеты отдела "IT", отслеживать приоритеты и выявлять обращения с задержкой обработки.

Ваша задача как RPA-разработчика — автоматизировать работу с данными о тикетах через **DataFrame** в **Puzzle RPA**, отфильтровать тикеты по отделу, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

---

**Шаги:**

1. **Создать исходный файл `tickets.xlsx` с 10 записями:**

```python
# tickets.xlsx
| ticket_id | user_name     | department   | priority | ticket_status | create_date |
|-----------|---------------|--------------|----------|---------------|-------------|
| T001      | "Иванов И."   | "IT"         | "высокий"| "открыт"      | 2026-03-10  |
| T002      | "Петров П."   | "продажи"    | "средний"| "открыт"      | 2026-03-09  |
| T003      | "Сидоров С."  | "IT"         | "низкий" | "в работе"    | 2026-03-08  |
| T004      | "Кузнецов К." | "маркетинг"  | "высокий"| "открыт"      | 2026-03-11  |
| T005      | "Морозова М." | "IT"         | "средний"| "закрыт"      | 2026-03-07  |
| T006      | "Новиков Н."  | "продажи"    | "низкий" | "открыт"      | 2026-03-10  |
| T007      | "Фролова Ф."  | "IT"         | "высокий"| "открыт"      | 2026-03-12  |
| T008      | "Смирнов С."  | "маркетинг"  | "средний"| "в работе"    | 2026-03-09  |
| T009      | "Егоров Е."   | "IT"         | "низкий" | "открыт"      | 2026-03-11  |
| T010      | "Лебедева Л." | "продажи"    | "высокий"| "закрыт"      | 2026-03-08  |
```

* **Типы данных:**

```python
ticket_id: str
user_name: str
department: str  # "IT", "продажи", "маркетинг", "поддержка"
priority: str  # "высокий", "средний", "низкий"
ticket_status: str  # "открыт", "в работе", "закрыт"
create_date: date
```

2. **Считать исходный файл `tickets.xlsx` в объект в DataFrame `tickets_df`**


3. **Добавить столбец `is_new` — флаг нового тикета (создан за последние 3 дня):**

```python
tickets_df["is_new"] = tickets_df["create_date"] > (today - 3)
```

4. **Отфильтровать тикеты отдела "IT":**

```python
it_tickets_df = FilterDataFrame(
    table=tickets_df,
    filter_condition='department == "IT"'
)
```

5. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="it_tickets.xlsx")

# Уведомляем пользователя
NotifyUser(message="Тикеты отдела IT успешно сохранены в it_tickets.xlsx!")
```

## **Часть 2**

После фильтрации тикетов отдела "IT" руководство хочет проанализировать данные о задержках обработки.   
Существует второй файл `departments.xlsx`, в котором указаны данные о подразделениях и времени обработки тикетов.   
Нужно объединить данные и выделить тикеты с задержкой более 2 дней, а результат сохранить и показать пользователю.

---

1. **Создать файл `departments.xlsx` с 8 записями:**

```python
# departments.xlsx
| ticket_id | department_name | ticket_delay | assigned_to   | resolution_date |
|-----------|-----------------|--------------|---------------|-----------------|
| T001      | "IT"            | 1            | "Техник А."   | 2026-03-11      |
| T002      | "продажи"       | 3            | "Менеджер Б." | 2026-03-12      |
| T003      | "IT"            | 4            | "Техник В."   | null            |
| T004      | "маркетинг"     | 2            | "Специалист Г."| 2026-03-13     |
| T005      | "IT"            | 1            | "Техник А."   | 2026-03-08      |
| T007      | "IT"            | 3            | "Техник В."   | null            |
| T009      | "IT"            | 1            | "Техник А."   | 2026-03-12      |
| T010      | "продажи"       | 2            | "Менеджер Б." | 2026-03-10      |
```

* **Типы данных:**

```python
ticket_id: str
department_name: str
ticket_delay: int  # задержка обработки в днях
assigned_to: str   # ответственный сотрудник
resolution_date: date  # дата решения (null если не решено)
```

2. **Считать исходный файл `departments.xlsx` в объект в DataFrame `departments_df`**

3. **Объединить `it_tickets_df` и `departments_df` по `ticket_id` (inner join):**

```python
combined_df = JoinDataFrames(
    table1=it_tickets_df,
    table2=departments_df,
    keys1=["ticket_id"],
    keys2=["ticket_id"],
    join_type="inner"
)
```

4. **Выбрать тикеты с задержкой > 2 дней:**

```python
delayed_tickets_df = FilterDataFrame(
    table=combined_df,
    filter_condition='ticket_delay > 2'
)
```

5. **Сохранить результат в Excel и уведомить пользователя:**

```python
# Сохраняем Excel файл
SaveExcel(file_path="delayed_it_tickets.xlsx")

# Уведомляем пользователя
NotifyUser(message="Аналитика по тикетам с задержкой обработки сохранена в delayed_it_tickets.xlsx!")
```
