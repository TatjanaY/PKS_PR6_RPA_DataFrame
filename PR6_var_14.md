## **Часть 1**

Компания **TeamTracker** занимается мониторингом работы сотрудников службы поддержки.   
Ежедневно обрабатываются данные о задачах, назначенных сотрудникам: часть задач выполнена, часть — в работе, а часть — просрочена.  
Руководство хочет контролировать сотрудников отдела "поддержка", отслеживать количество выполненных задач и выявлять сотрудников с низкой продуктивностью.

Ваша задача как RPA-разработчика — автоматизировать работу с данными о сотрудниках через **DataFrame** в **Puzzle RPA**, отфильтровать сотрудников по отделу, добавить аналитические столбцы, сохранить результаты и уведомить пользователя.

## **Часть 2**

После фильтрации сотрудников поддержки руководство хочет проанализировать выполнение задач.   
Существует второй файл `tasks.xlsx`, в котором указано количество выполненных задач каждым сотрудником.   
Нужно объединить данные и выделить сотрудников, выполнивших менее 3 задач, а результат сохранить и показать пользователю.

---

```python
| E010   | 14             | 10              | 2026-03-10     |
```

* **Типы данных:**

```python
emp_id: str
tasks_assigned: int
tasks_completed: int
last_task_date: date
```

2. **Считать исходный файл `tasks.xlsx` в объект в DataFrame `tasks_df`**

3. **Объединить `support_employees_df` и `tasks_df` по `emp_id`:**

```python
combined_df = JoinDataFrames(
    table1=support_employees_df,
    table2=tasks_df,
    keys1=["emp_id"],
    keys2=["emp_id"],
    join_type="inner"
)
```

4. **Выбрать сотрудников с количеством выполненных задач < 3:**

```python
low_performance_df = FilterDataFrame(

)
```
